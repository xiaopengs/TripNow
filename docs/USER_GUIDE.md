## 🚀 快速开始

### 第一步：创建账本

1. 打开 TripNow，点击「创建新账本」
2. 输入账本名称（如：云南七日游）
3. 系统自动识别当地货币（或手动选择）
4. 选择账本皮肤主题
5. 点击「创建」

### 第二步：邀请成员

1. 进入账本详情页
2. 点击「邀请成员」
3. 生成邀请二维码或分享链接
4. 成员扫码加入后，可选择「认领身份」

### 第三步：开始记账

三种记账方式：
- **手动记账**：点击 FAB 按钮 → 手动记账
- **拍照识别**：点击 FAB 按钮 → 拍照识别
- **语音记账**：点击 FAB 按钮 → 语音记账

## 📒 创建账本

### 账本设置

| 字段 | 说明 | 示例 |
|------|------|------|
| 账本名称 | 为这次旅行起个名字 | 「云南七日游」 |
| 货币 | 默认根据定位自动识别 | 人民币、美元、欧元 |
| 皮肤 | 选择喜欢的主题 | 海洋蓝、沙漠橙、森林绿 |
| 预算 | 可选，设置旅行预算 | ¥10,000 |

### 影子成员系统

**什么是影子成员？**

影子成员是临时占位符，用于：
- ✅ 快速记账时添加不在群里的成员
- ✅ 避免因为等人而耽误记账
- ✅ 成员加入后可认领身份

**使用场景：**

```
场景：你在餐厅和小明、小红吃饭，但小红还没加入账本
操作：
1. 手动记账时，选择「影子成员」
2. 输入「小红」作为影子成员
3. 小红加入账本后，可认领这个影子成员
4. 所有相关账单自动关联到小红
```

## 💰 记账功能

### 方式一：手动记账

**步骤：**
1. 点击底部橙色 FAB 按钮
2. 选择「手动记账」
3. 填写信息：
   - 标题（如：午餐）
   - 金额（如：¥150）
   - 类目（餐饮/交通/住宿/娱乐/购物/门票）
   - 分摊方式（平均分摊/自定义比例）
   - 支付人
4. 点击「保存」

**智能分摊：**
- **平均分摊**：所有成员平分
- **自定义比例**：指定每人分摊比例
- **公账支付**：从公账池中扣除

### 方式二：拍照识别

**步骤：**
1. 点击 FAB 按钮 → 拍照识别
2. 拍摄小票或账单
3. AI 自动识别：
   - 商家名称
   - 总金额
   - 类目
   - 置信度
4. 确认信息后保存

**识别准确率：**
- 清晰小票：95%+
- 手写账单：80%+
- 低置信度会进入「待处理收件箱」

### 方式三：语音记账

**步骤：**
1. 点击 FAB 按钮 → 语音记账
2. 说出账单信息
   - 示例：「今天午餐花了 150 块」
3. AI 解析语音内容
4. 确认后保存

**支持的语言：**
- 中文（普通话）
- English

### 待处理收件箱

**位置：** 首页右上角红点提示

**功能：**
- 显示置信度低于 70% 的 AI 识别结果
- 支持重新编辑、删除、重试
- 空状态时显示「所有账单已处理」

## 🏦 公账管理

### 什么是公账？

公账是一个**公共资金池**，用于：
- 💰 预收成员的钱（如：每人先交 ¥2000）
- 🧾 支付公共费用（如：门票、住宿）

### 公账操作

**充值公账：**
1. 进入「公账」页面
2. 点击「充值」
3. 输入金额和支付人
4. 确认充值

**公账支付：**
1. 记账时，选择「使用公账支付」
2. 金额从公账余额中扣除
3. 不计入个人分摊

**公账流水：**
- 查看所有充值记录
- 查看所有公账支出
- 实时余额显示

## 🤝 智能结算

### 结算算法

TripNow 使用**最小路径结算算法**：
- ⚡ 最少转账次数
- 🎯 对冲欠款关系
- 💸 降低转账金额

**示例：**

```
原始欠款关系：
A 欠 B  100 元
B 欠 C  100 元
C 欠 A  100 元

智能结算后：
无需转账！（三人欠款对冲为 0）
```

### 结算流程

1. 进入「结算」页面
2. 查看当前欠款关系
3. 点击「生成结算方案」
4. 查看优化后的转账方案
5. 点击「发起结算」
6. 生成微信支付二维码
7. 收款人扫码收款
8. 确认收款完成

### 结算通知

**微信支付二维码：**
- 点击结算卡片 → 生成收款二维码
- 支付方扫码 → 自动填充金额
- 收款方确认 → 更新账本状态

**结算提醒：**
- 可设置自动结算阈值（如：欠款超过 100 元提醒）
- 可设置结算周期（如：每日/每周提醒）

## ⚙️ 财务设置

### 抹零设置

**功能：** 对金额进行取整处理

**选项：**
- **不抹零**：保留原始金额（默认）
- **取整**：去小数（12.8 → 12）
- **四舍五入**：标准舍入（12.4 → 12, 12.5 → 13）

**使用场景：**
- 跨境旅行（避免汇率小数）
- 朋友间小额账单（简化计算）

### 汇率缓存

**功能：** 减少汇率查询次数

**设置：**
- **启用缓存**：减少网络请求
- **过期时间**：1-168 小时
- **手动刷新**：清除缓存，立即更新

**默认配置：**
- 缓存时间：24 小时
- 自动过期

### 自动结算

**功能：** 自动提醒结算

**配置：**
- **结算阈值**：欠款超过多少元提醒（默认 ¥100）
- **提醒频率**：每日/每周/手动

## ❓ 常见问题

### Q1: 如何处理多人支付一笔账单？

**A:** 记账时，选择「自定义分摊」，可以指定：
- 多个支付人
- 每人支付金额
- 每人分摊比例

---

### Q2: 影子成员认领后，历史账单会更新吗？

**A:** 会！认领影子成员后：
- ✅ 所有相关账单自动关联
- ✅ 欠款关系实时更新
- ✅ 结算方案自动重新计算

---

### Q3: AI 识别错误怎么办？

**A:** 两种处理方式：
1. **低置信度**：自动进入「待处理收件箱」，可重新编辑
2. **已保存**：在账单列表中找到，点击编辑修改

---

### Q4: 公账余额不足会怎样？

**A:** 
- 支付时提示「公账余额不足」
- 自动切换为「个人支付」
- 仍可使用公账部分余额

---

### Q5: 如何导出账单数据？

**A:** 当前版本支持：
- 📋 复制账单列表（Markdown 格式）
- 📸 截图分享
- 🔄 数据导入/导出（开发中）

---

### Q6: 支持离线记账吗？

**A:** 是的！
- ✅ 本地缓存所有数据
- ✅ 离线可查看账单
- ✅ 恢复网络后自动同步（需要后端服务）

---

### Q7: 数据安全吗？

**A:** 我们的安全措施：
- 🔒 本地存储加密
- 🛡️ 传输过程 HTTPS 加密
- 🚫 不上传敏感信息到第三方
- 📱 数据完全由用户掌控
