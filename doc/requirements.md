# TripNow 项目需求文档

## 核心目标
实现从"零门槛创建"到"AI/手动双轨记账"，最后"智能无感结算"的全闭环。

---

## 一、账本创建与管理 (The Genesis)
设计意图：消除用户开启记账时的心理负担。

### 1.1 创建入口
- 首页为空时：中心显示巨大的 (+) 新建账本
- 侧边栏：支持多账本管理，可切换历史账本（已归档）和进行中账本

### 1.2 创建表单字段
- **账本名称**：如"曼谷大吃大喝之旅"
- **默认币种**：调用定位 API
  - 境外 → 自动设为当地货币
  - 国内 → 默认为人民币
- **视觉皮肤**：提供 6 种低饱和度纯色背景

### 1.3 影子成员预设 (Shadow System)
- 在创建页下方点击 + 添加成员
- **逻辑**：管理员直接输入人名（如"大壮"、"二狗"）
  - 系统生成 ID，标记为 `is_claimed = false`
- **认领交互**：真实用户通过微信链接进入时，强制弹窗："认领身份：你是大壮还是二狗？"
  - 认领后，历史记账归属权自动迁移至该用户 UID

---

## 二、交互核心：扇形弹出菜单 (The FAB Fork)
设计意图：在 1 秒内响应用户的不同记账意图。

### 2.1 按钮状态
- 右下角固定悬浮按钮 (+)

### 2.2 扇形展开 (Fan-out)
- 点击后 (+) 旋转 45°
  - **左侧**：手动记一笔 (Manual) → 呼起半屏录入面板
  - **右侧**：拍照/OCR (Camera) → 呼起全屏相机
- **动画参数**：200ms 弹性曲线（Spring Curve），带有触感震动反馈

---

## 三、记账路径 A：AI 拍照识别 (Snap & Scan)
设计意图：利用 AI 彻底消灭重复劳动。

### 3.1 极速捕获
- 快门按下即入库，不等待 OCR 结果
- 图片在后台异步处理

### 3.2 统一类目对齐
AI 自动根据识别内容归类至：
- 餐饮、交通、住宿、门票、购物、其他

### 3.3 待处理收件箱 (Inbox)
- 首页按钮右上角显示红点数字，提示有待确认的 AI 账单
- **卡片交互**：显示缩略图 + AI 识别金额 + 自动选定的类目
- **手势**：右滑确认（入账），左滑删除（废片）

### 3.4 OCR 异常处理
- 若金额不清晰（置信度<0.6），金额栏变为红色 "?"，点击自动呼起键盘纠错
- 若无小票，AI 识别图像内容（如"烤鱼"），自动生成备注

---

## 四、记账路径 B：手动记一笔 (Manual Zen)
设计意图：解决路边摊、现金交易等无票据场景。

### 4.1 半屏模态窗
- 不跳转页面，从底部滑出，保持上下文感知

### 4.2 核心组件
- **金额键盘**：大号字体，自带 + - 简易运算
- **5+1 磁贴**：六个大图标类目，选中态带有呼吸灯效果
- **智能分摊**：
  - 默认"全员均分"
  - 点击头像可排除某人，或输入具体金额/份额
- **备注/图片**：支持语音转文字备注，或手动从相册选图补传

---

## 五、结算中心：最小路径算法 (Smart Settlement)
设计意图：解决朋友间"谈钱伤感情"的尴尬。

### 5.1 智能对冲
- **算法逻辑**：系统计算全局债权
  - A欠B 50，B欠C 50 → 生成 A直接给C 50 的建议

### 5.2 结算通知
- 生成一张"请款卡片"，包含明细及微信支付二维码

### 5.3 财务设置
- **抹零开关**：开启后，建议转账金额自动四舍五入到元

---

## 六、用户全链路路径覆盖 (Full Journey Coverage)

### 场景 1：创建账本
用户 A 登录 → 点击 新建账本 → 定位日本，币种选 JPY → 手动输入影子成员"大壮"

### 场景 2：第一笔开销 (AI)
落地机场，A 购买 3 张车票 → 点击 + → 拍照 → 锁屏走人
- 后台 AI 识别 3000 JPY → 归类 交通 → 分摊给 A 和 大壮

### 场景 3：成员加入 (认领)
大壮扫码进入账本 → 点击"我是大壮" → 瞬间看到自己欠 A 1500 JPY

### 场景 4：补充开销 (手动)
晚上吃章鱼烧无发票 → 大壮点击 + → 手动记一笔 → 输入 500 JPY，选 餐饮

### 场景 5：行程总结与结算
旅行结束 → A 点击 结算中心 → 系统根据所有 AI 和 手动记录，生成最终建议：大壮给 A XX 元

## 七、开发质量保证清单 (Final Checklist)

- [ ] **数据幂等性**：防止弱网下连点产生多笔重复账单
- [ ] **多币种汇率表**：系统每日更新一次汇率缓存，记账时固化当时汇率
- [ ] **本地缓存策略**：无网络时，记账数据必须存在 wx.setStorage 中，网络恢复后静默同步
- [ ] **计算精度**：后端所有涉及钱的计算必须使用 BigDecimal 或分（整数）进行，严禁使用浮点数
