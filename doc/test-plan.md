## 🎯 测试目标

1. 验证 FAB 扇形菜单交互是否正常
2. 验证 Inbox 收件箱功能是否完整
3. 验证记账流程是否顺畅
4. 验证数据持久化是否有效
5. 发现并记录所有问题

## 🧪 测试环境

### 硬件要求
- 设备：手机或桌面浏览器（移动端模拟）
- 系统：iOS / Android / macOS / Windows

### 软件要求
- Node.js >= 18
- 浏览器：Chrome / Safari / Edge
- 网络：需要连接 Google Gemini API

### 环境变量
```bash
# .env 文件
GEMINI_API_KEY=your_api_key_here
```

## 🔧 测试步骤

### 步骤 1：环境准备
```bash
# 1. 进入项目目录
cd /Users/ray/.openclaw/workspace/project/TripNow

# 2. 安装依赖
npm install

# 3. 配置环境变量
echo "GEMINI_API_KEY=your_key" > .env

# 4. 启动开发服务器
npm run dev

# 5. 访问地址
http://localhost:3000
```

### 步骤 2：功能测试
按测试用例顺序逐个执行：
1. FAB 菜单测试（FAB-001 ~ FAB-008）
2. Inbox 测试（INB-001 ~ INB-010）
3. 记账流程测试（EXP-001 ~ EXP-006）
4. 首页测试（DSH-001 ~ DSH-005）
5. 流水测试（REC-001 ~ REC-003）
6. 结算测试（SET-001 ~ SET-003）

### 步骤 3：问题记录
发现问题时，记录到 `doc/bug-tracker.md`：
- 问题描述
- 复现步骤
- 预期结果
- 实际结果
- 截图（如有）
- 优先级

### 步骤 4：修复验证
修复问题后，重新执行相关测试用例，确保问题已解决。
